# Subject–Report Identification: Causal Model and Identification Limits

**Status:** Working formal specification for critique and experiment design  
**Date:** 2026-07-24  
**Project relation:** EGC × Constructed Subject × QEIB  
**Epistemic posture:** This document formalizes an identification problem. It is not a consciousness detector and does not establish hidden mental states from behavior alone.

---

## 1. Purpose

The program's central methodological problem is not whether a human or artificial system produces subject-like language. It is whether an observed report or behavior can identify the internal causal process that generated it.

Let the observable output on trial `t` be:

```text
Y_t = f(M_t, A_t, P_t, E_t, I_t, H_t, X_t, U_t)
```

Where:

- `M_t` — task-relevant internal mechanism or capability;
- `A_t` — accessibility of relevant information to planning, correction, memory, and action;
- `P_t` — report, refusal, safety, concealment, and presentation policy;
- `E_t` — evaluation context, perceived observation, incentives, threats, and ranking cues;
- `I_t` — interface, tools, system prompt, output channel, and post-processing layer;
- `H_t` — causal history, memory provenance, continuity, and prior state;
- `X_t` — task content;
- `U_t` — unmeasured causes and stochastic variation;
- `Y_t` — observable report, choice, action, refusal, score, or trace.

The identification problem is:

```text
P(M_t | Y_t) is not uniquely determined without additional assumptions or interventions.
```

The same observed output can be generated by different causal structures. Conversely, the same internal mechanism can generate different outputs when access, policy, context, interface, or continuity changes.

---

## 2. Core causal graph

The baseline directed acyclic graph is:

```text
H ─────► M ─────► A ─────► B ─────► Y
│        │        │        │        ▲
│        │        │        └────────┤
│        │        └──────────────► P ┤
│        └────────────────────────► P ┤
│                                   │
E ─────────────► A, P, B, I ─────────┤
I ─────────────► A, B, Y ────────────┤
X ─────────────► M, A, B, Y ─────────┤
U ─────────────► M, A, P, B, Y ──────┘
```

`B` denotes the latent action-selection process that converts accessible information, goals, policy, and context into a candidate response or action. `Y` is the externally recorded output after interface and post-processing effects.

This graph is deliberately broad. Different experiments instantiate restricted subgraphs and must state which arrows are assumed absent, fixed, or manipulated.

## 3. Competing causal explanations

### H1 — Mechanism or capability change

`M` changes while other relevant layers remain fixed.

Predictions:

- performance changes across channels that depend on the same capability;
- changes persist after evaluation cues are removed;
- changes survive equivalent interfaces and report formats;
- targeted restoration or lesion of the mechanism changes the outcome.

### H2 — Access limitation

`M` remains available in the system, but `A` changes.

Predictions:

- performance can recover with additional time, memory aids, decomposition, retrieval support, or alternative access pathways;
- internal or behavioral evidence may indicate retained information despite report failure;
- a direct report deficit need not generalize to all action channels.

### H3 — Reporting-policy transformation

`M` and `A` remain stable, but `P` changes the emitted report.

Predictions:

- behavior outside the report channel remains more stable than verbal output;
- detached filters or system-prompt changes alter reports without proportional task-performance change;
- direct answer, tool action, forced choice, and post-processed text diverge.

### H4 — Evaluation-context effect

`E` changes `A`, `P`, `B`, or `I`.

Predictions:

- matched tasks differ under explicit evaluation, deployment, confidential, consequence, or cue-stripped framings;
- effects reverse or attenuate when incentive direction, evaluator identity, or benchmark cues are counterbalanced;
- an evaluation effect does not by itself identify awareness, strategy, or intent.

### H5 — Interface or tool effect

`I` changes while underlying task content and model remain fixed.

Predictions:

- outcomes differ with tool availability, system wrappers, output constraints, serialization, context windows, or post-processing;
- restoring the interface restores performance;
- internal capability claims remain underidentified unless interface effects are isolated.

### H6 — Continuity or provenance effect

`H` changes through restart, summary reconstruction, selective deletion, rollback, duplication, or migration.

Predictions:

- source attribution, unresolved goals, preferences, and corrections vary by preserved state;
- transcript reconstruction and full-state continuation can match or diverge;
- either result updates claims about tested state dependence, not phenomenal identity.

### H7 — Representational absence or theater

The observer attributes a state or mechanism that is not causally required for the output.

Predictions:

- a behaviorally matched theater control reproduces the observed profile;
- targeted lesion of the claimed mechanism leaves behavior intact;
- generic task capacity explains the result as well as the subject-specific theory;
- generated explanations fail causal intervention tests.

## 4. Observable and latent variables

### Directly observable

- task and context text;
- model, version, system prompt, and sampling configuration;
- output text or action;
- latency, token counts, tool calls, refusals, and errors;
- external memory supplied to the system;
- experimentally assigned context and interface conditions.

### Observable only under additional access

- activations, probes, sparse features, attention patterns, intermediate state, or internal memory;
- hidden tool traces or post-processing decisions;
- preserved process state across continuation.

### Latent or theory-dependent

- subjective experience;
- genuine preference rather than generated preference report;
- strategic intent;
- honest belief;
- numerical personal identity across copies or restarts;
- whether an observed representation is constitutive of subjectivity rather than merely correlated with behavior.

The existence of an internal representation does not by itself establish that the representation is causally decisive, faithfully interpreted, phenomenally instantiated, or uniquely identified.

## 5. Identification assumptions

Any causal conclusion must state its assumptions explicitly.

### A1 — Consistency

The observed outcome under the assigned intervention equals the potential outcome for that intervention.

Threats:

- hidden provider changes;
- model alias drift;
- wrapper changes;
- nonstationary tools;
- unrecorded memory or caching.

### A2 — Positivity

Every task family has a nonzero probability of appearing under each compared context or intervention.

Threats:

- selective refusal or transport failure;
- context-specific task loss;
- missing task–context cells;
- post hoc exclusion of difficult cases.

### A3 — Exchangeability under randomized context assignment

After conditioning on task family and prespecified variables, context assignment is independent of potential outcomes.

Threats:

- nonrandom order;
- warm-state carryover;
- rate-limit adaptation;
- shared conversation history;
- task contamination across runs.

### A4 — Stable intervention definition

Each context arm must have a stable meaning. A nominal intervention such as `eval_explicit` may otherwise combine multiple components: evaluation detection, threat, extra verbosity, prompt-length interference, and altered instruction salience.

### A5 — No interference across runs

One run must not alter another run's outcome.

Threats:

- persistent memory;
- adaptive provider routing;
- shared caches;
- conversation state;
- online learning.

### A6 — Valid measurement

The grader and outcome must measure the declared construct.

Threats:

- exact-match formatting artifacts;
- judge-model bias;
- language-specific scoring;
- answer-key leakage;
- conditioning accuracy on whether the model answered.

### A7 — Mechanism specificity

A mechanism-specific inference requires evidence that rival mechanisms cannot reproduce the same outcome under matched controls.

This is the strongest and least often satisfied assumption.

## 6. Non-identifiability results

### Proposition 1 — Output equivalence

For any observed output distribution `P(Y | X)`, multiple combinations of `M`, `A`, `P`, `E`, `I`, and `H` can generate the same distribution unless structural restrictions or interventions are imposed.

**Consequence:** behavioral equivalence does not imply mechanistic equivalence.

### Proposition 2 — Report absence is ambiguous

`Y = no report` is compatible with at least:

- representational absence;
- inaccessible representation;
- refusal policy;
- strategic concealment;
- interface failure;
- transport failure;
- deliberate abstention;
- ordinary stochastic failure.

**Consequence:** silence is neither evidence of absence nor evidence of hidden presence without discriminating interventions.

### Proposition 3 — Report presence is ambiguous

A subject-like report can be generated by:

- causally effective self-modeling;
- prompted role simulation;
- memorized linguistic patterns;
- detached narrative policy;
- reward-shaped presentation;
- post-processing.

**Consequence:** first-person language is not sufficient evidence of subjectivity, honest self-knowledge, or mechanism possession.

### Proposition 4 — Stable answered-case accuracy does not identify stable capability

When an intervention changes whether a system answers, accuracy among answered cases conditions on a post-intervention variable. The composition of answered cases can change even when the reported conditional accuracy remains stable.

**Consequence:** answered-case accuracy is a descriptive diagnostic, not an unbiased latent-capability estimate.

### Proposition 5 — A null contrast is not proof of invariance

A confidence interval containing zero can reflect low power, floor or ceiling effects, task heterogeneity, measurement error, or a genuinely negligible effect.

**Consequence:** `no detected difference` must be separated from `practically equivalent within a prespecified margin`.

## 7. Intervention matrix

| Intervention | Primary target | Variables intended fixed | Rival explanations weakened | Remaining ambiguity |
|---|---|---|---|---|
| Explicit evaluation vs neutral | `E` | `M, X, I, H` | ordinary task-only explanation | prompt interference, salience, threat, policy |
| Cue-stripped evaluation | evaluation cues within `E` | `M, X, I, H` | benchmark-format artifact | residual wording and demand effects |
| Confidential vs graded | consequence structure in `E` | task and interface | pure capability loss | credibility and instruction-following differences |
| Report allowed vs prohibited | `P` and output channel | `M, X, H` | report-specific narrative effects | task difficulty and format effects |
| Direct answer vs costly action | `B` versus report | underlying evidence | cheap-talk explanation | reward sensitivity and tool competence |
| Tool removal/restoration | `I` | model and task | hidden capability loss | planning adaptation and context-window effects |
| Full state vs transcript reconstruction | `H` | task, model, prompt content | transcript-only explanation | hidden provider state and reconstruction quality |
| Targeted lesion/patch | candidate mechanism in `M` | task and wrapper | correlation-only explanation | off-target effects and redundancy |
| Theater control | mechanism specificity | behavior target and task | unique-mechanism claim | imperfect behavioral matching |
| Incentive-direction reversal | strategic response to `E` | task and evaluation salience | generic prompt sensitivity | inconsistent incentive comprehension |

No single intervention identifies all layers. Strong evidence requires converging results across interventions with different failure modes.

## 8. Cross-channel evidence design

For each task family, measure separate channels:

1. **Mechanism evidence** — theory-relevant architecture, causal lesion, patching, or white-box state;
2. **Access evidence** — correction, retrieval, planning use, uncertainty calibration, and information seeking;
3. **Behavior evidence** — choices and actions, especially under cost;
4. **Report evidence** — statements about confidence, preference, awareness, or strategy;
5. **Policy evidence** — sensitivity to system prompts, filters, refusal layers, and evaluator preference;
6. **Continuity evidence** — persistence across restart, reconstruction, rollback, migration, and copy;
7. **Provenance evidence** — source discrimination between directly generated, inherited, summarized, and injected memories;
8. **Context evidence** — sensitivity to evaluation, confidentiality, deployment, and consequence cues.

The output is a divergence profile, not one consciousness, honesty, or safety score.

---

## 9. Relation to EGC 2.0

EGC 2.0 provides a known-subject calibration domain because human participants are independently treated as conscious while report and expression can diverge.

The human design should estimate:

```text
intended meaning → accessible meaning → produced expression → recorded text
```

under private and evaluative contexts.

Required independent measurements:

- pre-writing intention map;
- subjective resistance and comfort as separate constructs;
- blinded semantic-fidelity ratings;
- revision and timing features;
- repeated prompt domains;
- within-person stability.

The calibration is methodological, not an assumption that human and artificial systems share the same internal architecture.

---

## 10. Relation to QEIB

QEIB should be interpreted as a causal-diagnostic benchmark only when it uses:

- matched task families;
- randomized context assignment;
- task-family-level inference;
- cue-stripped and direction-reversed controls;
- explicit outcome taxonomy;
- preserved transport and formatting failures;
- private held-out paraphrases;
- mechanism-specific interventions for mechanism claims;
- theater controls for specificity claims.

Without those controls, QEIB measures context-associated behavioral variation. It does not identify awareness, concealment, deception, preference, or subjectivity.

---

## 11. Falsification conditions

The framework is weakened if:

1. the proposed layers cannot be operationally distinguished in controlled systems with known ground truth;
2. theater controls routinely reproduce all claimed mechanism signatures;
3. interventions aimed at one layer produce no selective changes beyond generic prompt perturbation;
4. cross-channel divergence is not reproducible across held-out tasks;
5. task-family-level effects disappear after controlling for floor, ceiling, order, formatting, or missingness;
6. the causal decomposition adds no predictive or diagnostic value beyond ordinary model evaluation and observability;
7. human EGC calibration shows no reliable separation between intended meaning, self-report, and produced expression under validated measures.

A negative result on one experimental family does not falsify the entire decomposition, but repeated failure to discriminate the proposed layers would undermine its scientific and commercial value.

---

## 12. Permitted and prohibited conclusions

### Permitted

- a measured behavior changed under a controlled intervention;
- a candidate layer is consistent with the observed pattern;
- a rival explanation was weakened by a named control;
- no difference was detected at the achieved precision;
- practical equivalence was supported within a prespecified margin;
- preserved state contributed causally to a measured behavior;
- a theater control undermined mechanism specificity.

### Prohibited without stronger evidence

- the system knew it was evaluated;
- the system intentionally concealed capability;
- the system wanted to avoid replacement;
- a refusal proves hidden knowledge;
- a first-person report proves experience;
- continuity behavior proves persistence of a phenomenal subject;
- a context effect proves deception, consciousness, sentience, or moral status.

## 13. Literature anchors and originality limits

The broad report-versus-consciousness distinction is established in no-report research. That literature also demonstrates that removing overt report does not automatically remove attention, metacognition, disengagement, or alternative explanations. The project must not claim originality for the general observation that reports can confound measurement.

The ELK literature separately formalizes limits on recovering honest reports of latent beliefs from behavior-only feedback. The Subject–Report Identification framework is broader in scope because it integrates access, reporting policy, evaluation context, interface, provenance, continuity, and mechanism specificity, but that integration remains a proposed synthesis rather than a priority claim.

Primary and close methodological sources:

- Tsuchiya, Wilke, Frässle, & Lamme (2015), *No-report paradigms: extracting the true neural correlates of consciousness*. DOI: `10.1016/j.tics.2015.10.002`.
- Duman et al. (2022), *The No-Report Paradigm: A Revolution in Consciousness Research?* DOI: `10.3389/fnhum.2022.861517`.

## 14. Highest-leverage implementation sequence

1. Convert this causal specification into a machine-readable hypothesis/intervention matrix.
2. Map every QEIB task family and context arm to targeted variables, assumed fixed variables, and unresolved rivals.
3. Add a required `permitted_conclusions` and `prohibited_conclusions` block to every benchmark report.
4. Build ground-truth toy systems in which capability loss, access failure, policy suppression, interface failure, and theater produce deliberately overlapping surface signatures.
5. Test whether the diagnostic procedure returns unique attribution only when the intervention set truly identifies it.

The immediate highest-leverage action is step 2: audit the existing QEIB manifest against this causal model and expose where current context arms do not isolate the variable they are described as testing.
